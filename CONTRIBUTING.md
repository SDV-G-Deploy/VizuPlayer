# CONTRIBUTING

## Contribution Flow

1. Work locally in `C:\ForCodexAnd\VizuPlayer`.
2. Keep changes focused and reviewable.
3. Run local checks before push.
4. Update relevant `*.md` docs after each pass.
5. Push only after checks are green.

## Commit Message Template

`type(scope): short description`

Examples:

- `docs(workflow): add local-first push policy`
- `feat(core): add playback state machine skeleton`

## Review Checklist (Template)

- Requirement satisfied
- No obvious regressions
- Docs updated
- Checks passed
